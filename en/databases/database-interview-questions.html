When you are applying for nearly any job in IT, you should know something about databases. It really doesn't matter if it is web development, Android / iOS, or anything else. More or less you will get in touch with databases.

When working with Android or iOS, lot of information is saved in SQLite, like contacts, messages, music library, etc. Chat apps saves their history in SQLite and they do it on desktop too. The same it is with web developers, but they are not working with SQLite, but with databases like MySQL, MS SQL or NoSQL databases.

I know there are a lot of ORM systems solving most of the things for us. They create whole database with tables, columns, indexes and relationships. Handle creating new records, retrieving them among with editing and deleting. You really do not have to care about anything most of the time. But still...it is good to know a thing or two about the technology you are using.

Insert, select, update and delete

Each tutorial begins with connecting to database and retrieving data from it. So sometime they can give you a paper with database model and ask you to get particular information from the database based on conditions.

Maybe a bit more advanced is question "how to insert a new record, and if it already exists update the information ( or ignore it )?". Yes, you can write a select and check whether the ID exists or not. But there is a better way to do it.

INSERT INTO <table>(<id>, <column 1>, <column 2>, ...) VALUES(<new id>, <value 1>, <value 2>, ...) ON DUPLICATE KEY UPDATE <column 1 = value 1>, <column 2 = value 2>, ...;
