Do you copy the whole project ( or modified files only ) everytime you change something? Are there many things you have to care about when deoploing new version?

Deployment process can be very easy, but it can get more complex fast. You start to use Composer or some CSS preporcessor and there it is. Before it can be run, you have to download all dependencies and create CSS files. Doing this all the time will drive you crazy.

There are lot of things you may want from a deploy script, so I divide this to more blog posts, each about different part of deploy process. E.g.:

  • clone specific branch from GitHub
  • check files with PHP lint
  • run Composer and if there are tests available, run them too
  • check javascript with JS lint
  • install Node modules and run grunt / gulp / webpack
  • create new version tag
  • copy files to server
