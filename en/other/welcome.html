Programming theory

I think this is the most important theme. The syntax of a language one learns quickly, then just a little logical thinking and he is able to solve problems and produce usable applications. But...is that enough? For simple tasks and small projects yes, but once it comes to more complex staff, teamwork and so on, then it is different story. The code of this "programmer" is hard to read, mostly bad documented ( or not documented at all ) and it is complicated to extend it.

Everyone certainly met with a situation where he found his own code intricate and complicated. That it is the time to move a bit further again and start to use proven practices to some common issues and also think about the style of writing the code.

If you stop learning, then the projects you work on are stuck in whatever time period you decided to settle.
